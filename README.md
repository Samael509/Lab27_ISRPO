## Главные выводы
1. ASP.NET Core позволяет создать рабочий веб-сервер буквально в 4 строки кода
2. Middleware выполняется до маршрутов и позволяет обрабатывать каждый запрос централизованно
3. C# автоматически сериализует объекты в JSON при возврате из маршрута

## Ответы на вопросы
1. **Minimal API vs MVC:** Minimal API - это упрощённый подход без контроллеров и классов, всё прямо в Program.cs. MVC разделяет логику на Model, View, Controller - удобно для больших проектов.
2. **MapGet до Use:** Middleware не выполнится для этих маршрутов - порядок регистрации важен, middleware должен быть выше маршрутов.
3. **PascalCase → camelCase:** Это стандарт JSON (RFC 8259) и соглашение JavaScript. ASP.NET Core по умолчанию использует `JsonNamingPolicy.CamelCase` при сериализации.
4. **HTTP коды:** 200 - OK (успех), 401 - Unauthorized (нет доступа), 404 - Not Found (не найдено).
5. **dotnet run vs watch:** `dotnet run` запускает один раз, `dotnet watch run` автоматически перезапускает при изменении файлов.
